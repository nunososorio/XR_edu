## 9. Collection Management Workflow

### Creating a Collection

1. **Name:** Use clear, descriptive names
   - ✅ "Science: Physics Simulations (Grade 10)"
   - ❌ "Physics 10"

2. **Description:** Include helpful metadata
   - Target age/grade
   - Primary use case
   - Update frequency
   - Curator information

3. **Permissions:** Decide sharing model
   - Private (single user)
   - Team/Organization (selected members)
   - Public (read-only sharing)
   - Collaborative (invite-based editing)

4. **Tags:** Apply consistent tagging
   - Platform tags
   - Subject tags
   - Pedagogical approach
   - Age/grade level

### Organizing within Collections

**Best Practices:**
- Use alphabetical or pedagogical ordering
- Include date added for recent resources
- Mark "quality-verified" items
- Highlight "recommended" resources
- Note seasonal or event-based content

### Collection Maintenance

**Regular Tasks:**
- Verify links monthly (linkrot detection)
- Update resource descriptions
- Add new items
- Remove outdated resources
- Refresh accessibility info

**Seasonal Collections:**
- Holiday-themed content
- Subject-specific units
- Grade-level transitions
- Professional development cycles

## 10. Import/Export Data

### Export to CSV

Template for exporting collection data:

```csv
title,url,description,platform,subject,age_min,age_max,cost_type,tags,verified,date_added
"Anatomy Lab VR","https://example.com","Interactive anatomy","quest-3","Science: Biology","13","18","Free","science,biology,interactive,vr-app","true","2026-03-10"
```

### Import Resources

Prepare CSV file with required columns:
- title (required)
- url (required)
- description
- platform
- subject
- age_min, age_max
- cost_type
- tags (comma-separated)
- accessibility_notes
- duration_minutes
- pedagogical_goal

---

## Usage Tips

1. **Start with main categories** - Don't create too many nested collections initially
2. **Use consistent naming** - Helps with search and organization
3. **Tag generously** - Tags enable flexible organization independent of hierarchy
4. **Include metadata** - Descriptions help others understand content
5. **Regular review** - Periodically check for obsolete or broken links
6. **Collaborate** - Invite colleagues to contribute and curate together
7. **Share selectively** - Use permissions to control audience while gathering feedback
